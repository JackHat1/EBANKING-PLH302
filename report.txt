    3)Προδιαγραφές:

        α) Προσβαση στο συστημα(Login):

                i) Ταυτοποίηση χρήστη με Username , Password (Τι βελτίωση θα μπρορεί να εχει)

                ii) Αν αποτύχουν 3 προσπάθειες -> Restricted mode (επικοινωνια με τραπεζα για ξεκλειδωμα/ Ξεκλειδωμα απο admin)

                iii) Δυνατότητα αλλαγής Password
                iv)  Ειδοποίηση αλλαγής Password στον ένα χρόνο απο την τελευταία αλλαγή

        β) Τραπεζικοί Λογαριασμοί

                i) Κάθε τραπεζικός λογαριασμός αναγνωρίζεται μοναδικά μέσω ενός IBAN μήκους 27 χαρακτήρων, το οποίο δημιουργείται αυτόματα από το σύστημα. Η δομή του IBAN περιλαμβάνει:

                    -> Κωδικό χώρας: 2 χαρακτήρες

                    -> Κωδικό είδους λογαριασμού: 3 ψηφία (100 για προσωπικό, 200 για επιχειρηματικό)

                    -> Αριθμό λογαριασμού: 15 ψηφία

                ii) Ο κάθε λογαριασμός έχει έναν βασικό κάτοχο, που μπορεί να είναι φυσικό πρόσωπο ή επιχείρηση, ανάλογα με τον τύπο του λογαριασμού.

                iii) Σε κάθε λογαριασμό εφαρμόζεται ετήσιο επιτόκιο δανεισμού. Οι τόκοι υπολογίζονται σε ημερήσια βάση αλλά πιστώνονται στο τέλος κάθε μήνα.

                iv) Οι προσωπικοί λογαριασμοί δίνουν τη δυνατότητα ύπαρξης και δευτερευόντων κατόχων, οι οποίοι ωστόσο πρέπει να είναι αποκλειστικά φυσικά πρόσωπα.

                v) Μια επιχείρηση μπορεί να διατηρεί αποκλειστικά έναν επιχειρηματικό λογαριασμό.

                vi) Οι επιχειρηματικοί λογαριασμοί έχουν σταθερό μηνιαίο τέλος διατήρησης (Maintenance Fee), το οποίο καταλογίζεται στο τέλος κάθε μήνα και εισπράττεται από την τράπεζα.

                vii) Το υπόλοιπο ενός λογαριασμού μπορεί να μεταβληθεί μόνο μέσα από την εκτέλεση συναλλαγών (transactions).

                viii) Η ίδια η τράπεζα διατηρεί έναν δικό της επιχειρηματικό λογαριασμό, στον οποίο συγκεντρώνονται τα τέλη και από τον οποίο καταβάλλονται οι τόκοι.   

        γ) Συναλλαγές

                i) Κάθε συναλλαγή διαθέτει έναν μοναδικό κωδικό, ο οποίος δημιουργείται αυτόματα από το σύστημα.

                ii)Για κάθε συναλλαγή είναι απαραίτητο να είναι γνωστός ο διεκπεραιωτής (Transactor). Αυτός μπορεί να είναι:

                    -> ένας πελάτης,

                    -> ένας διαχειριστής,

                    -> ή η ίδια η τράπεζα (όταν η συναλλαγή αποτελεί αποτέλεσμα αυτοματοποιημένης διαδικασίας).

                iii) Μια συναλλαγή, ανάλογα με το είδος της, μπορεί να αφορά έναν ή δύο τραπεζικούς λογαριασμούς.

                iv) Κάθε συναλλαγή πρέπει να συνοδεύεται τουλάχιστον από μία αιτιολογία (π.χ. «Κατάθεση σε ATM»). Όταν εμπλέκονται περισσότεροι λογαριασμοί, τότε απαιτούνται ξεχωριστές αιτιολογίες:

                    -> για τη χρέωση (αποστολέας)

                    -> και για την πίστωση (παραλήπτης).

        δ) Λογαριασμοί Πληρωμής

                i) Κάθε λογαριασμός διαθέτει έναν μοναδικό κωδικό πληρωμής RF, ο οποίος συνδέει τη συνδρομή ενός πελάτη με την επιχείρηση που εκδίδει τον λογαριασμό.

                ii) Εκτός από τον κωδικό πληρωμής RF, ο λογαριασμός περιέχει:

                    -> τον μοναδικό αριθμό λογαριασμού (κωδικός έκδοσης),

                    -> το ποσό που πρέπει να καταβληθεί,

                    -> τον εκδότη του λογαριασμού,

                    -> τις ημερομηνίες έκδοσης και λήξης,

                    -> καθώς και οποιαδήποτε άλλη πληροφορία κρίνεται αναγκαία.

                iii) Μια επιχείρηση μπορεί να εκδώσει πολλαπλούς λογαριασμούς με τον ίδιο κωδικό πληρωμής RF, όμως κάθε φορά μόνο ένας από αυτούς μπορεί να είναι ενεργός μέσα στο σύστημα.

        ε) Πάγιες Εντολές

                i) Κάθε πάγια εντολή έχει μοναδικό κωδικό, τίτλο, περιγραφή και είναι ενεργή εντός ενός συγκεκριμένου χρονικού διαστήματος.

                ii) Στις εντολές πληρωμής λογαριασμών, υποδεικνύεται από το δημιουργό τους το μέγιστο ποσό λογαριασμού που μπορεί να πληρωθεί αυτόματα.

                iii) Στις εντολές μεταφοράς υποδεικνύεται από το δημιουργό τους η συχνότητα εκτέλεσης (σε μήνες), η συγκεκριμένη ημέρα εκτέλεσης, καθώς και το ποσό μεταφοράς.

                iv) Οι πάγιες εντολές εκτελούνται αυτόματα από το σύστημα σε προκαθορισμένη ημερομηνία.
        
                    -> Αν πρόκειται για εντολή πληρωμής λογαριασμού, την ημερομηνία λήξης του λογαριασμού.
        
                    -> Αν πρόκειται για εντολή μεταφοράς, την υποδειχθείσα από την εντολή ημέρα.

                v) Εκτελεστής μιας πάγιας εντολής θεωρείται η ίδια η τράπεζα.

                vi) Η εκτέλεση μιας πάγιας εντολής μπορεί να υπόκειται σε τέλος το οποίο χρεώνεται αυτόματα στον δημιουργό της εντολής (δημιουργός της πάγιας εντολής) και καταγράφεται στην τράπεζα ως ξεχωριστή συναλλαγή.

                vii) Μια πάγια εντολή εκτελείται σε περιοδικές απόπειρες. Ενδεικτικά δίνονται τα εξής παραδείγματα:
        
                    -> Η πληρωμή του λογαριασμού ΔΕΗ του 1ου διμήνου 2025 (απόπειρα εκτέλεσης Εντολής Πληρωμής Λογαριασμού)
        
                    -> Η μεταφορά χρημάτων για την πληρωμή ενοικίου Απριλίου 2025 (απόπειρα εκτέλεσης Εντολής Μεταφοράς)

                viii) Αν η απόπειρα εκτέλεσης μιας πάγιας εντολής αποτύχει για 3 φορές, τότε το σύστημα αποθηκεύει πληροφορίες για την αποτυχία, και σταματά τη συγκεκριμένη απόπειρα. Η εντολή ωστόσο συνεχίζει να υφίσταται κανονικά.
        
                    -> Αν πρόκειται για πάγια εντολή εξόφλησης λογαριασμού, πρέπει να αγνοείται ο συγκεκριμένος Αριθμός Λογαριασμού αλλά όχι άλλοι που ενδέχεται να έχουν εκδοθεί με τον ίδιο κωδικό πληρωμής RF
        
                    -> Αν πρόκειται για περιοδική εντολή Μεταφοράς Χρημάτων θα πρέπει να αγνοηθεί η συγκεκριμένη απόπειρα μεταφοράς, αλλά όχι οι επόμενες.

                ix) Η εκτέλεση των παγίων εντολών γίνεται κατά προτεραιότητα με βάση την παλαιότητά τους ως προς την προγραμματισμένη ημερομηνία εκτέλεσης τους. Δηλαδή μια εντολή που ήταν να εκτελεστεί στις 10/4 έχει μεγαλύτερης προτεραιότητας από μια που ήταν να εκτελεστεί στις 11/4.

                x) Η εκτέλεση μιας πάγιας εντολής γίνεται με τη δημιουργία μιας ή περισσότερων συναλλαγών και η εκτέλεση αυτών των συναλλαγών οδηγεί στη δημιουργία κινήσεων για τους εμπλεκόμενους λογαριασμούς. Δείτε ενδεικτικά τη σχετική ροή εργασιών και πληροφορίες στο Σχήμα 2 (εφόσον η απόπειρα εκτέλεσης ολοκληρωθεί).
                
            ζ) Simulate Time Passing :
